# 一页纸总览（Overview）

## 背景与目标

我们希望开发一个“抖音视频小助手”，按照用户配置的**主题关键词**与**时间计划**，自动（或半自动）完成：

- 浏览/搜索公开视频内容
- 依据策略做筛选与排序（质量、相关性、账号画像等）
- 对候选视频执行互动动作（点赞/收藏/关注/评论），并全量留痕可追溯

## 设计原则（硬约束）

- **合规优先**：尊重平台服务条款与法律法规，不做绕过风控/反自动化的功能设计。
- **人机协同**：默认“建议 + 待确认”，高风险动作（尤其评论/私信）强制人工确认或二次校验。
- **可控速与可回滚**：所有动作有速率上限、配额与熔断；任何策略发布可回退；支持一键暂停。
- **可审计**：所有输入、决策、执行、结果、失败原因都有日志与证据（截图/HTML快照/回放）。
- **可扩展**：支持多主题、多账号、多任务并行；策略与执行引擎解耦。

## 范围与非目标

### 范围（第一阶段）
- 主题管理：关键词/标签/排除词/语言/地域偏好
- 任务调度：按时间窗口执行（例如每天 10:00-11:00）
- 内容获取：站内搜索 + 推荐流浏览（以“合规可得数据”为准）
- 筛选与评分：规则 + 可选 AI 评分（文本/封面/标题/基础指标）
- 执行动作：点赞、收藏（低风险）；关注（中风险需配额）；评论（高风险默认人工确认）
- 留痕与报表：今日执行、成功率、风险命中、内容与动作明细

### 非目标（明确不做）
- 绕过验证码/风控/反自动化检测
- 批量刷量、虚假互动、欺诈引流
- 抓取私密数据或账号敏感信息

## 关键成功指标（KPI）

- **覆盖度**：每个主题每日有效候选内容数
- **效率**：从内容发现到互动执行的平均耗时
- **质量**：互动后 24h 的正反馈（回关/点赞/互动率）作为代理指标
- **稳定性**：任务成功率、失败重试率、风控触发率
- **安全与合规**：违规/封禁/投诉为 0（或可量化阈值内）

## 推荐实现路径（结论先行）

短期推荐：**“浏览器扩展/桌面自动化（执行层） + 本地/服务端策略与调度（控制层）”**的组合模式：

- 执行层尽量贴近“真实交互环境”（浏览器），并把高风险动作放到人工确认流程里
- 控制层负责策略、排队、配额、审计、监控，保证可控与可追溯
