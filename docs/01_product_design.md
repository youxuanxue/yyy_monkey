# 产品方案（Product Design）

## 1. 目标用户与使用场景

### 目标用户
- **内容运营/投放**：需要持续跟进某些赛道（母婴/美妆/本地生活/教育等）并做高质量互动
- **品牌市场/BD**：寻找达人/爆款素材，做收藏、关注、后续触达
- **个人创作者**：定时跟进热点话题，互动拉近关系（但必须“慢、稳、真”）

### 典型场景
- **场景 A：主题巡检**：每天固定时间检索“关键词 + 排除词”，收藏潜在爆款
- **场景 B：高质量互动**：对高相关视频点赞+收藏，并将评论草稿推给人工确认
- **场景 C：账号拓展**：对符合画像的创作者关注并记录原因（用于后续分组运营）

## 2. 核心概念与信息架构

### 概念
- **主题（Topic）**：关键词、标签、排除词、语言/地域偏好、内容类型偏好
- **任务（Job）**：主题 + 时间计划 + 执行策略 + 配额
- **候选内容（Candidate）**：抓取/发现到的视频条目（含证据与元数据）
- **策略（Policy）**：筛选规则、评分模型、动作组合、阈值与黑白名单
- **动作（Action）**：点赞/收藏/关注/评论（含状态机、失败原因、证据）
- **审计记录（Audit Log）**：所有决策与执行的可追溯记录

### 视图与模块（UI/控制台）
- **主题管理**：新建/编辑主题；导入关键词；黑白名单（账号/话题）
- **任务调度**：时间窗口、频次、并发、配额、熔断规则
- **候选池**：按评分排序；一键加入“待执行队列”；批量标注“不相关”
- **执行队列**：实时状态；暂停/继续；失败重试；人工确认队列
- **评论工作台**：AI 生成草稿 + 人工编辑 + 审核通过后提交
- **报表**：按主题/任务统计；转化代理指标；风险命中统计

## 3. MVP（最小可行版本）

### MVP 功能清单（4-6 周）
- 主题：关键词/排除词/地域/语言（基础）
- 调度：每日/每周时间窗口；任务开关；每小时/每日动作配额
- 获取：站内搜索结果列表采集（公开可见信息）+ 推荐流浏览（可选）
- 筛选：规则（关键词命中、账号黑名单、发布时间范围）；基础评分
- 动作：
  - 点赞（自动，低风险，受配额/限速控制）
  - 收藏（自动，低风险）
  - 关注（中风险，默认关闭或低配额）
  - 评论（自动提交，**受限额度**，详见“评论自动化策略”）
- 留痕：每条候选视频保存“来源、时间、截图/快照、动作结果（含评论内容）”
- 审核机制：默认保守；评论自动化开启后仍保留“抽检/复核”入口与一键暂停

### MVP 不做
- 大规模自动评论（仅允许小额度、强约束的自动评论）
- 批量多账号并行（先单账号打通闭环）
- 复杂画像与预测模型（先规则与基础 LLM 评分）

## 4. 交互策略（可配置）

### 动作组合（示例）
- **保守模式**：只收藏（用于素材沉淀）
- **互动模式**：点赞 + 收藏（默认间隔随机 20-60 秒）
- **拓展模式**：对高评分账号“关注”（每日上限极低）
- **评论模式（受限自动）**：命中严格条件 → 自动提交 1 条短评论（每日上限极低）；其余进入草稿/人工确认

### 评论草稿策略（示例）
- 以“内容细节”与“真诚问题”提升自然度
- 禁止广告/引流/联系方式；敏感词过滤
- 支持“品牌语气/人设模板”

### 评论自动化策略（MVP 约束）
- **额度/速率**：
  - 按账号配置评论速率上限：**每分钟不超过 2 条**
  - 配合冷却时间（comment cooldown），避免短时间密集提交
- **候选范围**：允许对所有候选放开（不强制白名单），但命中黑名单直接禁止
- **模板白名单**：只允许使用预先审核通过的短模板（可带少量变量，如“视频里提到的点”）
- **安全过滤**：敏感词/引流词/广告词/联系方式模式命中则拒绝提交
- **证据与回放**：提交前后截图 + 实际提交文本入审计
- **抽检与熔断**：按比例抽检人工复核；触发风控信号/失败率异常立即暂停评论能力

## 5. 权限与账号体系

### 账号与角色（建议）
- Owner：配置主题、策略、配额、接入
- Operator：处理候选池与评论工作台
- Auditor：只读审计与报表

### 多账号（第二阶段）
- 每个抖音账号独立配额与风险阈值
- 账号级黑白名单与隔离（避免交叉污染）

## 6. 运营与客服（面向内部/企业）

- 任务异常告警：失败率上升、疑似风控、登录失效
- 变更审计：策略/配额变更需要审批（可选）
- 证据留存：用于复盘与争议处理
