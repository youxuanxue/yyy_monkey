# 里程碑与交付计划（Roadmap）

## 阶段 0：对齐与预研（1 周）

- 明确业务边界：允许的动作范围与默认配额（保守）
- 明确数据边界：可采集字段、证据留存方式、存储周期
- 风险评审：合规/隐私/安全评审通过
- 技术预研：选择执行载体（插件优先），跑通“搜索→候选提取→留痕”

**交付物**
- 本仓库 `docs/` 方案评审通过
- Demo：采集候选列表并保存截图/快照

## 阶段 1：MVP 闭环（4–6 周）

### 1.1 控制平面（后端/本地服务）
- 主题/任务配置（CRUD）
- 调度触发（时间窗口）→ 生成 run
- 候选入库、动作队列、状态机、失败重试（保守）
- 审计日志与证据引用（可追溯）

### 1.2 执行平面（浏览器插件优先）
- 搜索与浏览：按主题关键词抓取候选
- 证据：候选截图/快照；动作前后截图
- 低风险动作：点赞/收藏（自动，可关闭，受配额/限速控制）
- 评论能力（两种模式二选一或并存）：
  - 默认：评论工作台（生成草稿 + 人工确认）
  - 可选：受限额度自动评论（模板白名单 + 强过滤 + 抽检 + 熔断）

### 1.3 运营视图
- 候选池列表：筛选/排序/标注不相关
- 执行队列：暂停/恢复/重试/查看失败原因
- 报表：按主题/任务汇总（候选数、执行数、成功率、失败类型）

**验收标准**
- 全链路可跑通：主题→run→候选→动作→回执→报表
- 可控：配额、熔断、一键暂停有效
- 可审计：任意动作可回放证据并解释原因
 - 自动评论（若开启）：满足门禁约束，且能一键下线模板与暂停评论
 - 自动评论速率限制（若开启）：每账号每分钟提交不超过 2 条（硬限制），超限拒绝并可审计

## 阶段 2：稳定性与规模化（4–8 周）

- 执行器工程化：适配页面变化、健壮性提升（断点续跑、幂等）
- 多账号支持：账号隔离、配额隔离、权限体系
- Worker 化（可选）：Playwright 执行器作为独立 worker 接入队列
- 告警完善：登录失效/失败率突增/队列积压/存储异常

**验收标准**
- 任务成功率显著提升，异常可自动止损
- 多账号并行仍保持审计一致性与配额可控

## 阶段 3：策略与增长（持续迭代）

- 筛选/评分升级：更强规则引擎、可选 AI 相关性评分
- 评论模板与人设：多模板 A/B 测试（仍强制审核或抽检）
- 数据闭环：互动后代理指标复盘，反哺策略（保守）

## 研发组织与职责（建议）

- CEO/PM：需求边界、KPI、合规红线、发布节奏
- Tech Lead：架构、接口抽象、工程质量与可观测性
- FE：插件/控制台
- BE：调度/队列/审计/报表
- QA：回放、幂等、异常注入、回归
- Compliance/Sec：隐私、安全、风控准入
